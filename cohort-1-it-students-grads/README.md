# Career Paths Overview – Cohort 1 (IT Students & Grads)

As a student or recent graduate, you don’t need to pick a lifelong specialty yet, but it helps to understand the main **entry-level** paths you can aim for in cybersecurity operations.

This guide gives you a quick overview of common roles, what they actually do day‑to‑day, and how they typically progress over the first few years.

---

## 1. SOC Analyst (Security Operations Center)

**Typical entry titles:**  
- SOC Analyst (L1 / Tier 1 / Junior)  
- Cybersecurity Analyst L1  
- Security Monitoring Analyst

**What you do day‑to‑day:**  
- Monitor alerts from SIEM/EDR and other security tools.  
- Triage alerts: decide what’s noise vs what’s real.  
- Collect basic evidence (logs, screenshots) and escalate real incidents.  
- Follow playbooks for common scenarios (phishing, brute‑force, malware).

**Good fit if you enjoy:**  
- Puzzles and pattern recognition.  
- Fast feedback and working in shifts.  
- Log analysis and tooling.

**Early career path (0–5 years):**  
- Tier 1 SOC Analyst → Tier 2 SOC Analyst (investigations & response) → Tier 3 / Threat Hunter / Detection Engineer → SOC Lead / IR roles.

---

## 2. Incident Response (IR) & DFIR

**Typical early titles:**  
- Junior Incident Responder  
- DFIR Analyst (Associate)  
- CSIRT Analyst

**What you do day‑to‑day:**  
- Investigate security incidents in more depth.  
- Analyse hosts, logs, and sometimes memory or disk images.  
- Contain attacks (isolate systems, revoke access) and help with remediation.  
- Document what happened and lessons learned.

**Good fit if you enjoy:**  
- Forensic “detective” work and timelines.  
- Deep technical debugging.  
- High‑stakes incidents (on‑call at times).

**Early career path (0–5 years):**  
- SOC / Support / IT → Junior DFIR / IR Analyst → Incident Responder / DFIR Consultant → Senior IR / DFIR lead.

> As a student, DFIR is usually a second step: many people start in SOC or IT and then move into DFIR after 1–2 years.

---

## 3. Cloud Security (Blue Team)

**Typical early titles:**  
- Cloud Security Analyst (Junior)  
- Security Engineer – Cloud (Associate)  
- Cloud Operations / DevOps with security focus

**What you do day‑to‑day:**  
- Review and harden cloud accounts (IAM, networking, storage).  
- Enable and monitor cloud logging and security services.  
- Work with dev/ops teams on secure configurations and guardrails.  
- Investigate cloud‑specific alerts and misconfigurations.

**Good fit if you enjoy:**  
- Working with AWS/Azure/GCP and IaC.  
- Automation and scripting.  
- Designing and improving systems.

**Early career path (0–5 years):**  
- IT / Cloud / DevOps → Cloud Security Analyst → Security Engineer (Cloud) → Security Architect / Platform Security roles.

---

## 4. GRC (Governance, Risk & Compliance)

**Typical early titles:**  
- Security / Risk Analyst (Junior)  
- Compliance Analyst  
- Information Security Analyst (GRC focus)

**What you do day‑to‑day:**  
- Help maintain policies, standards, and procedures.  
- Assist with risk assessments and control checks.  
- Support audits and customer/security questionnaires.  
- Translate technical issues into business and risk language.

**Good fit if you enjoy:**  
- Writing, documentation, and structured thinking.  
- Talking with non‑technical stakeholders.  
- Understanding how organisations work.

**Early career path (0–5 years):**  
- Junior GRC Analyst → Security/Risk Analyst → GRC Specialist / Security Program Manager → potentially CISO track over time.

---

## 5. Red Team / Offensive Security (Longer‑Term)

**Typical titles (usually NOT first roles):**  
- Junior Penetration Tester  
- Red Team Operator  
- Security Consultant (Offensive)

**What you do day‑to‑day:**  
- Perform authorised attacks to test defences.  
- Write detailed reports on findings and fixes.  
- Collaborate with blue‑team/engineering to improve security.

**Why it’s usually not truly “entry‑level”:**  
- Many teams expect 1–3 years of IT or blue‑team experience first.  
- You need a solid understanding of how systems are built and defended.[

As a student, you can still **prepare** by doing CTFs, labs, and bug bounties, but be open to starting in SOC/IT first.[

---

## Example Early Pathways for Students

Here are some realistic 3–5 year paths you might follow:

1. **Student → SOC → IR/DFIR**  
   - Year 0–1: Intern / graduate SOC analyst.  
   - Year 1–3: Tier 2 analyst, start doing deeper investigations.  
   - Year 3–5: Move into dedicated IR / DFIR consulting or internal CSIRT.[

2. **Student → IT / Cloud → Cloud Security**  
   - Year 0–1: IT support, junior sysadmin, or junior cloud engineer.  
   - Year 1–3: Take on more security tasks (hardening, logging, reviews).  
   - Year 3–5: Move into Cloud Security Engineer / Security Analyst roles.[

3. **Student → GRC → Security Program Roles**  
   - Year 0–1: Junior risk/compliance analyst.  
   - Year 1–3: Own parts of an ISO/NIST program, vendor reviews, assessments.  
   - Year 3–5: Security program / GRC specialist or manager.

---

## How to Use This Overview in Your Plan

In your 30‑60‑90 plan and intake questions, use this overview to:

- Pick **1–2 roles** that sound interesting (not 5 at once).  
- Note what you like about them (tools, tasks, work style).  
- Align your labs and projects with those roles:
  - SOC interest → log analysis, SIEM labs, alert triage.  
  - DFIR interest → forensic, timeline, and incident walkthrough labs.  
  - Cloud interest → cloud lab, misconfig + fix projects.  
  - GRC interest → risk register, policy writing, and control mapping.

You can change your mind as you learn more. The goal in your first 90 days is not to lock in a lifetime career, but to get close enough to **try a path in reality** and build experience you can talk about in your first applications.
