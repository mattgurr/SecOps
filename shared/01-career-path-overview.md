# Cybersecurity Career Paths – Short Overview

Use this to quickly pick a starting lane for your 90‑day plan.

---

## Cohort 1 – IT Students & Grads

- **Good starting roles:** SOC / Security Analyst (junior), IT support with security tasks, junior GRC or risk roles.
- **Common 3–5 year paths:**
  - Student → SOC → IR/DFIR or threat hunting.
  - Student → IT/Cloud → Cloud Security / Security Engineer.
  - Student → Junior GRC → GRC specialist / program roles.
- **Focus now:** Build foundations, do labs, and create 1–2 small projects aligned to SOC, DFIR, cloud, or GRC.

---

## Cohort 2 – Career Changers

- **Good starting roles:** SOC / Security Analyst, GRC / Risk / Compliance Analyst, or IT / cloud roles as a stepping stone.
- **Common 3–5 year paths:**
  - Career changer → SOC → L2 / investigations → IR.
  - Career changer → GRC → security program / risk lead.
  - Career changer → IT/Cloud → Cloud Security / Security Engineer.
- **Focus now:** Map your past career to security tasks, pick one main path (SOC, GRC, or Cloud), and build 2–3 artifacts that tell your transition story.

---

## Cohort 3 – Existing IT Professionals

- **Good starting moves:** Add security to your current role (tickets, hardening, logging, CI/CD checks, risk work) and make it visible.
- **Common 3–5 year paths:**
  - Helpdesk → SOC Analyst.
  - Sysadmin / Network / Cloud → Security / Cloud Security Engineer.
  - Developer / DevOps → AppSec / Product Security.
  - IT PM / BA / Service Mgmt → GRC / security program roles.
- **Focus now:** Use your 90‑day plan to “own” a small slice of security work where you are, document impact, then leverage that for an internal or external move.

---

## Quick Decision Hints (All Cohorts)

- Like **dashboards, alerts, and puzzles** → Start with SOC / blue team.  
- Like **writing, policies, and talking to the business** → Start with GRC / risk.  
- Like **cloud, infra, and automation** → Aim for cloud/security engineering.  
- Like **forensics or hacking** → Plan a longer route via SOC or IT first.
