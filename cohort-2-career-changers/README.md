# Career Paths Overview – Cohort 2 (Career Changers)

You’re not starting from zero. As a career changer, your previous experience is a major **asset** – the key is mapping it to realistic cybersecurity paths and building enough proof that you can do the work.

This guide outlines common paths that work well for people coming from non‑security backgrounds, and how your existing skills can feed into them.

---

## 1. SOC / Security Analyst (Blue Team)

**Typical titles for career changers:**
- SOC Analyst (L1 / Tier 1)
- Cybersecurity Analyst
- Information Security Analyst

**Why it’s career‑changer friendly:**
- Many SOC teams hire for attitude, communication, and analytical thinking, then train specific tools.
- Shift work and structured playbooks can make it easier to learn while you do the job.

**Day‑to‑day work:**
- Monitor alerts and tickets from SIEM/EDR tools.
- Triage alerts, decide what’s important, and escalate accurately.
- Gather evidence and update incident tickets and documentation.

**Good backgrounds for this path:**
- Customer service / call centre → strong communication and triage.
- Operations / logistics → process‑driven, good under pressure.
- Finance / healthcare → used to regulated environments.

**Early path (0–5 years):**
- SOC Analyst L1 → SOC Analyst L2 / Investigator → Threat Hunter / Detection Engineer → SOC Lead / IR roles.

---

## 2. GRC, Risk & Compliance

**Typical titles:**
- Information Security / GRC Analyst (Junior)
- Compliance Analyst
- Risk Analyst

**Why it’s career‑changer friendly:**
- Heavy emphasis on communication, documentation, and understanding business context.
- Less requirement for deep technical skills at the start (though they help).

**Day‑to‑day work:**
- Maintain policies, standards, and procedures.
- Assist with risk assessments and vendor/security questionnaires.
- Help prepare for audits and evidence collection.

**Good backgrounds for this path:**
- Business / finance / accounting → risk thinking and governance.
- Legal / compliance → policies, regulation, contracts.
- Teaching / consulting → explaining complex topics simply.

**Early path (0–5 years):**
- Junior GRC / Compliance Analyst → Security/Risk Analyst → GRC Specialist / Program Manager → potential CISO/Head of Security track.

---

## 3. Cloud Security via IT / Cloud / DevOps

For many career changers, the practical route is: **move into IT or cloud first, then into security** – especially if you currently have no technical background.

**Typical stepping‑stone titles:**
- Service Desk / IT Support
- Junior Sysadmin / Network Technician
- Junior Cloud Engineer / DevOps

**Security‑flavoured target roles:**
- Cloud Security Analyst
- Security Engineer (Cloud)
- SecDevOps / Platform Security Engineer

**Day‑to‑day work (target roles):**
- Harden cloud environments (IAM, network, storage).
- Enable and monitor logging and cloud security services.
- Work with dev/ops teams on secure patterns and guardrails.

**Good backgrounds for this path:**
- Any role where you’re comfortable learning tools and processes.
- People comfortable with scripting, automation, or data workflows.

**Early path (0–5 years):**
- Move into IT / Cloud role → take on more security tasks → Cloud Security Analyst / SecEng → Architect/Lead.

---

## 4. DFIR & Offensive Security (Longer‑Term Targets)

**DFIR (Digital Forensics & Incident Response)** and **Red Team / Pen Testing** are attractive, but often better treated as second‑step goals for career changers.

**Why usually second step:**
- Employers typically expect hands‑on IT or SOC/blue‑team experience first
- Requires deeper technical skills in OS internals, networks, and tooling.

Suggested path:
- Career changer → SOC / IT / GRC → then specialise in DFIR or offensive after 1–3 years, using labs, certs, and projects to demonstrate skill.

---

Use these questions to narrow your path:

- Do you like **talking with people and writing**? → Start with GRC, risk, or security analyst roles with a business focus.
- Do you like **tools, dashboards, and puzzles**? → SOC / blue‑team roles are a strong fit.
- Do you like **systems and automation**? → Aim for IT / cloud / DevOps as a step into cloud security.
- Do you like **deep technical troubleshooting**? → Long‑term tilt toward DFIR or red team, via a blue‑team or IT step first.

---

## How This Connects to Your 90‑Day Plan

Within your 90‑day cohort‑2 plan, you will:

- Pick **one primary path** (SOC, GRC, or Cloud) to aim at first.  
- Align your labs and projects with that path.  
- Use your previous career stories as proof you can handle similar problems in a security context.

You can pivot later as you learn more; the goal for the first 90 days is to become a **credible junior candidate** for at least one realistic path, not to lock in a forever specialisation.
